## 🚀 How to Use

### Quick Start for Client Setup (3 steps)
1. Build: `build_desktop_exe.bat` → creates `dist/PyPondo.exe`
2. Copy: Move `PyPondo.exe` to client PC
3. Configure: Create `server_host.txt` with admin PC name/IP, then run

### Install on Client PC
```batch
# Option A: Using pre-built EXE
PyPondo.exe

# Option B: Using batch setup script
setup_client.bat

# Option C: Using PowerShell
.\setup_client.ps1

# Option D: Using Python directly
python -m pip install flask flask-sqlalchemy flask-login werkzeug
python desktop_app.py
```

### Test Connectivity
```batch
python test_client.py
```

### Debug Issues
```batch
set PYPONDO_VERBOSE=1
python desktop_app.py
```

## 📝 Configuration

### server_host.txt Format
Plain text file with admin server address:

```
# My admin PC
MY-ADMIN-PC
```

Or use IP:
```
# Admin server IP
192.168.1.100
```

### Environment Variables
```batch
set PYPONDO_SERVER_HOST=MY-ADMIN-PC
set PYPONDO_SERVER_PORT=5000
set PYPONDO_VERBOSE=1
```

## 🔧 Technical Highlights

### Hidden Subprocess Pattern
```python
# All subprocess calls now use:
subprocess.run(cmd, **hidden_subprocess_kwargs())
subprocess.check_output(cmd, **hidden_subprocess_kwargs())

# Function handles Windows-specific flags:
# - CREATE_NO_WINDOW
# - STARTUPINFO with STARTF_USESHOWWINDOW
# - SW_HIDE
# - Safe no-op on non-Windows
```

### Discovery Order
1. server_host.txt (explicit user config)
2. PYPONDO_SERVER_HOST env var
3. Network gateway (auto-detected)
4. Local network scan (net view)
5. Machine's own IPv4 addresses

### Error Messaging
- Console: Detailed text message
- GUI: Pop-up dialog with instructions
- Verbose: Shows all discovery attempts

## 💡 Usage Tips

1. **Best Method**: Use hostname in server_host.txt
   - Survives IP address changes
   - Works across network
   - Example: `MY-ADMIN-PC`

2. **Fallback**: Use IP address if hostname fails
   - Find IP: `ipconfig` on admin PC
   - Example: `192.168.1.100`

3. **Debug Issues**: Enable verbose mode
   - `set PYPONDO_VERBOSE=1` then run app
   - Shows all discovery attempts
   - Helps diagnose connectivity

4. **Test First**: Run connectivity test
   - `python test_client.py`
   - Checks config, network, HTTP
   - Provides clear pass/fail
